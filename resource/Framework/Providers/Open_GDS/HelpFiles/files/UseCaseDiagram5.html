Use Case Diagram

The Use Case Diagram shows use cases provided by the system and user categories which are called Actors.


Select desired mouse tool by clicking on the buttons on the top of the window. Use the context help to see the meaning of each mouse tool selection.


The use case diagram is used to clarify the system boundaries and show the functionality provided. This diagram is normally used during the requirements phase of development.


The elements of this diagram are:

- Use cases. These are user scenarios.

- Actors. These repesent user categories or roles.

- System. This lets you define which system that provides (implements) the different use cases.

See the context help for more information about these concepts.


First add a system to the diagram. This represents the system you are about to build. Then consider if the users of this system have different roles and thus could have different requirements on the system. Add an Actor for each role.

Then add use cases inside the system. Use the Association relationship to show which actor is using which use case.
