UML Modeller

Getting Started with Open GDS's UML Modeller

In this tool you work with five different diagram types. Each diagram you create is stored as a separate file.

Select File->New to create a new diagram.

Supported diagram types are:


Class Diagram

Package Diagram (Packages are created within class diagrams)

Sequence Diagram

Use Case Diagram

State Diagram


Getting help when and where it is needed

A Context Help is integrated in the tool. Press F1-key , or press to activate context help. The context help is designed to explain the different controls and it also includes a description of the notation of UML.

When viewing a diagram, the tool buttons for that diagram type are shown (tool buttons for Class Diagram are shown above). The descriptions of the UML notation is primarily located in the help texts for the tool buttons.

An overview of each diagram type can be seen by moving the mouse over the diagram selection menu (shown below).

All controls and dialogs have Context Help.

Forward and reverse engineering

These functions are available for class diagrams and state diagrams (only reverse engineering). Forward and reverse engineering is available in the Tools menu. To get a detailed description of these functions, select a class diagram and then select forward or reverse engineering in the Tools menu. Activate the Context Help (F1-key) for the dialogs which are opened. Reverse Engineering can be activated from within the project environment by right clicking on an lvclass or MyComputer.

Synchronizing class diagrams

Each class in a Class Diagram can be linked to the corresponding GOOP code. Right click on a class to bring up the property dialog and link a class to the code. When this is done you can synchronize on class level (right click on a class) or diagram (menu selection in the Tools menu).

Printing and exporting

Diagrams can be printed and also exported to .png or .jpg files. These operations are available in the File menu.
